# Small-Sample Learning for Defect Classification

Few-shot classification of semiconductor manufacturing defects from grayscale images, built for a challenge sponsored by Intel Corporation. The goal: learn to detect and classify defects accurately from a very small number of labeled examples per class, mimicking how a human expert learns from limited exposure to defect types.

> ⚠️ **Data note:** This repository does **not** include the Intel-provided dataset. The `data/` folder is a placeholder — see [Data](#data) below before running anything.

## Problem Statement

Real manufacturing lines produce highly imbalanced, scarce labeled data: most units are good, defects are rare, and new defect types appear faster than they can be manually labeled at scale. This project explores whether a pretrained computer-vision backbone, fine-tuned on only a handful of images per defect class, can reach production-relevant accuracy (~87%) and how quickly that accuracy is reached as labeled examples accumulate.

## Approach

- **Backbone:** EfficientNet-B0 pretrained on ImageNet, fine-tuned end-to-end. Transfer learning lets the model reuse general edge/texture features instead of learning them from scratch on a handful of images.
- **Input handling:** grayscale defect images are resized to 224×224 and replicated to 3 channels so they're compatible with the ImageNet-pretrained weights, then normalized with ImageNet statistics.
- **Augmentation:** random horizontal flip + small rotation (±10°) — enough to combat overfitting on tiny datasets without distorting defect geometry.
- **Classification head:** the original 1000-way ImageNet head is replaced with `Dropout → Linear(512) → ReLU → BatchNorm → Linear(num_classes)`.
- **Evaluation:** overall accuracy, per-class precision/recall/F1, and a confusion matrix on a held-out test split, plus a **few-shot learning curve** — accuracy vs. number of labeled images per class — to directly show learning speed as data grows, which is the metric this challenge cares about most.

## Data

Place your dataset under `data/` in `torchvision.datasets.ImageFolder` layout:

```
data/small_sample_20/
├── scratch/
│   ├── img001.png
│   └── ...
├── particle/
│   └── ...
└── good/
    └── ...
```

If your raw images are instead a flat folder with class-prefixed filenames (e.g. `scratch_0001.png`), use the helper in `src/dataset.py`:

```python
from src.dataset import restructure_flat_folder
restructure_flat_folder("data/small_sample_20")
```

The Intel-provided dataset used during development is proprietary and is intentionally excluded from this repository (see `.gitignore`). Swap in your own defect images with the same folder layout to reproduce the pipeline.

## Usage

**Command line (recommended for reproducible runs):**

```bash
python -m src.main --data-root data/small_sample_20 --epochs 20 --batch-size 32
```

This trains the model, saves loss/confusion-matrix plots and model weights to `results/`, and prints a full classification report for both the training and held-out test splits.

**Notebook (recommended for exploration / Colab):**

Open `notebooks/defect_classification.ipynb` and run top to bottom. It walks through the same pipeline interactively, including the few-shot learning-curve sweep in the final section.

## Results

The model targets ~85% classification accuracy on held-out data. The key evaluation artifact is the **few-shot learning curve** (`results/learning_curve.png`), generated by `src/learning_curve.py`, which retrains the model from scratch at increasing sample-size budgets (e.g. 2, 4, 8, 16, 32 images per class) and plots mean accuracy ± std against the target line. This makes learning speed — not just final accuracy — directly visible, which is the primary criterion this challenge is evaluated on.

## Key Skills Demonstrated

- Few-shot / small-sample learning for real-world computer vision
- Transfer learning with pretrained CNN backbones on domain-shifted (grayscale, industrial) imagery
- Handling class imbalance in industrial defect data
- Model evaluation and trade-off analysis (accuracy vs. label budget)
- System-level thinking about latency, scalability, and data constraints in semiconductor manufacturing settings

## Future Work

- Explore metric-learning / prototypical-network approaches for true k-shot classification rather than fine-tuning a shared classifier head.
- Add class-balanced sampling or focal loss for more severe imbalance.
- Benchmark lighter backbones (MobileNetV3, EfficientNet-Lite) for on-line inspection latency budgets.
